# Сайт подбора курьеров и сборщиков

Next.js, две сборки из одного кода:

- `npm run build:github` — статический экспорт для GitHub Pages (стенд, noindex, форма в demo-режиме)
- `npm run build:server` — server-таргет с API лидов и SQLite (прод, после покупки домена и VPS)
- `npm run dev` — локальная разработка на http://localhost:3100

## Переменные окружения (server-таргет)

Создайте `.env` по образцу:

| Переменная | Что это |
|---|---|
| `NEXT_PUBLIC_SITE_URL` | Прод-домен для canonical/OG/sitemap (до покупки пусто) |
| `NEXT_PUBLIC_METRIKA_ID` | Счётчик Яндекс Метрики |
| `TELEGRAM_BOT_TOKEN` | Бот уведомлений колл-центра (телефон в сообщении маскируется) |
| `TELEGRAM_CHAT_ID` | Чат колл-центра |
| `IP_HASH_SALT` | Соль хэширования IP в антифроде |
| `ADMIN_TOKEN` | Доступ к мини-админке лидов `/admin?token=...` |

## Структура

- `lib/data/` — сервисы, города, роли, тарифы (с источниками и датами), вопросы, глоссарий
- `content/blog/*.mdx` — статьи; `publishAt` в будущем = отложенная публикация (ежедневный cron в CI пересобирает стенд и публикует наступившие)
- `research/` — артефакты исследования: семантика, минус-лист для Директа, SERP-аудит, официальные тарифы, гео-частоты
- `app/api/` — приём лидов с антифродом (исключается из статической сборки автоматически)

## Лиды (server-таргет)

`POST /api/lead` — валидация, риск-скоринг (honeypot, скорость заполнения, rate-limit по IP-хэшу), дедупликация по телефону, сохранение в SQLite (`data/leads.db`), маскированное уведомление в Telegram. Статусы ведутся в `/admin?token=...`: новый → дозвон → квалифицирован → вышел на смену / отказ / фрод.
